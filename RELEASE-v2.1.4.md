# v2.1.4 - 作者活跃分提示修复版

## 修复

- 修复提示框移动到 `<body>` 后，渲染函数仍从日历根节点查找提示框，得到 `null` 并在鼠标悬停时触发 JavaScript 异常的问题。
- 提示框引用现在直接传入日历渲染函数，确保鼠标悬停和键盘聚焦都能正常显示。
- 修复鼠标悬停日期方块时，作者存在字数或发布明细便不显示个人活跃分的问题。
- 每位作者现在始终显示当天活跃分，并在分数后附带新增字数、修改字数、首次发布和更新发布明细。
- 修复同一作者同一天发布或编辑多篇内容时，在提示框中出现多条分散记录的问题。
- 同一作者当天的全部记录会合并，个人活跃分和创作明细按天汇总。
- 前端增加空作者列表兼容处理，避免不完整数据导致悬停事件报错。

## 显示示例

```text
Akagi_Zen：2264 活跃分（新增 664 字，发布 2 篇）
```

## 升级后验证

1. 安装并启用 2.1.4。
2. 打开包含活跃日历的公开页面。
3. 将鼠标移动到有活跃记录的日期方块。
4. 确认提示框先显示当天总分，然后逐位显示作者当天总分和创作明细。
5. 管理员可访问以下地址确认运行版本：

   ```text
   /apis/api.activity.foxbridge.team/v1alpha2/calendar/debug?year=2026&build=214
   ```

预期运行标识：

- `pluginVersion: 2.1.4`
- `apiVersion: v1alpha2`
- `buildSignature: hac-2.1.4-214`
