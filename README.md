# Halo Plugin Activity Calendar

> Halo 全站创作活跃度日历插件

以 GitHub Contributions Calendar 的形式，在 Halo 页面中展示所有作者每天的创作活跃度。

- 适配 Halo 2.25.0 及以上版本
- 作者：[Akagi_Zen](https://github.com/fhfzfy1231)
- 许可证：MIT

## 功能

- 统计文章和独立页面的有效新增字数、修改字数、首次发布和更新发布。
- 汇总所有后台作者的每日活跃度，分数越高，日期方块颜色越深。
- 鼠标悬浮或键盘聚焦日期时，显示当天活跃作者及每人的创作明细。
- 当前年份与往年切换，默认显示最近 5 年。
- 手机端横向滑动，自动适配深色主题。
- 在 Halo 默认编辑器的工具箱或 `/` 命令中直接插入。
- 可分别设置组件标题、主题颜色、显示年份数量和作者明细开关。
- 插件设置页可以调整各项评分权重和四档颜色阈值。

## 默认评分规则

```text
每日活跃分 =
新增有效字数 × 1
+ 修改有效字数 × 0.5
+ 首次发布内容 × 800
+ 更新已发布内容 × 200
```

自动保存不会被当成独立的活跃行为。插件比较相邻内容快照，只累计真实发生变化的字数。

默认颜色区间：

| 活跃分 | 颜色等级 |
| ---: | --- |
| 0 | 灰色 |
| 1–199 | 浅绿色 |
| 200–599 | 较浅绿色 |
| 600–1499 | 中绿色 |
| 1500 及以上 | 深绿色 |

## 安装

1. 下载 Release 中的 `plugin-activity-calendar-2.1.2.jar`。
2. 进入 Halo 后台的“插件”页面，点击右上角“安装插件”。
3. 上传 JAR，安装并启用。

## 页面中使用

1. 打开需要展示日历的文章或独立页面。
2. 在默认编辑器工具箱中选择“全站创作活跃日历”；也可以输入 `/` 后搜索“活跃日历”。
3. 在组件顶部调整标题、主色、年份数量和作者明细开关。
4. 保存并发布页面。

插件启用时会先把现有内容作为统计基线，避免把整篇旧文章误算成当天新增。安装后的编辑和发布活动会被记录；安装前的详细修改历史不会倒推补算。

## 本地构建

环境要求：JDK 21、Node.js 24、pnpm 10。

```bash
./gradlew build
```

构建后的插件位于 `build/libs/`。

## v1.0.1 更新

- 修复前台接口异常时导致活跃度数据加载失败的问题。
- 增强游客访问稳定性和空数据处理。
- 修复版本号并优化发布流程。

## v2.1.1 更新

- 修复历史基线统计在线程 `reactor-tcp-epoll` 上读取插件设置时触发阻塞异常的问题。
- 设置只在每次统计请求开始时异步读取一次，避免逐篇文章重复读取。
- 修复历史内容能够解析、但 `baselineRecordCount` 和 `totalScore` 始终为 0 的问题。
- 统一插件清单、调试接口和 Gradle 构建产物的版本号为 2.1.1。

## v2.1.2 更新

- 修复未登录访客访问 `/apis/**` 被 Halo 重定向到登录页，导致前台显示“活跃度数据加载失败”的问题。
- 日历数据改为由 Halo 服务端安全嵌入页面，前端不再依赖游客可访问的动态 API。
- 管理员调试接口迁移到 `v1alpha2`，避开旧插件类残留造成的路由冲突。
- 增加 `hac-2.1.2-212` 构建签名，便于确认实际运行的后端代码。
- 最近 10 年的数据一次扫描完成并缓存 15 秒，避免重复读取所有内容。
